# Kamilyan Playbook

## Project Vision

Kamilyan is a transformation advisory firm that helps organizations move from uncertainty to clarity.

Rather than leading with implementation, Kamilyan helps executive leaders understand their current state, define a future vision, and develop a practical roadmap for transformation.

The website should reflect thoughtful leadership, structured thinking, and practical guidance—not simply advertise consulting services.

---

# Brand Positioning

## Core Promise

**Transformation starts with clarity.**

Kamilyan helps organizations make confident technology and transformation decisions before investing in implementation.

## Positioning Statement

Kamilyan provides advisory services focused on assessment, strategy, and planning. When organizations are ready to execute, implementation can be delivered through Blue Ridge Web Tech (BRWT).

## Relationship to BRWT

Kamilyan and BRWT are complementary brands.

**Kamilyan**

- Strategy
- Advisory
- Assessments
- Future-state planning
- Transformation roadmaps

**BRWT**

- Engineering
- Implementation
- Delivery leadership
- AI implementation
- Managed services
- Technical execution

Kamilyan answers **"What should we do?"**

BRWT answers **"How do we build it?"**

# The Kamilyan Advisory Framework

## 1. Current-State Audit

**Executive Question**

Where are we today?

Purpose:

Assess technology, architecture, delivery maturity, organizational capabilities, technical debt, and AI readiness.

---

## 2. Commerce Platform & Capability Assessment

**Executive Question**

Do we have the right foundation?

Purpose:

Evaluate platform fit, scalability, business alignment, modernization opportunities, and capability gaps.

---

## 3. Future-State Strategy

**Executive Question**

Where do we want to go?

Purpose:

Define the future vision, operating model, customer experience goals, target capabilities, AI opportunities, and strategic direction.

---

## 4. Transformation Roadmap

**Executive Question**

How do we get there?

Purpose:

Translate strategy into a practical, prioritized roadmap for execution.

---

## 5. Execution (via BRWT)

Engineering, implementation, delivery leadership, and operational execution.

# Design Principles

The website should feel:

- Premium
- Executive
- Modern
- Calm
- Confident
- Structured
- Practical

Avoid:

- Flashy effects
- Excessive animation
- Generic consulting clichés
- Heavy shadows
- Busy layouts

Whitespace is a design element.

Typography should carry much of the visual hierarchy.

---

# UX Principles

Every section should answer one question.

Hero:  
Why am I here?

Problem:  
Why does this matter?

Framework:  
How do you help?

Services:  
What do I receive?

About:  
Why should I trust you?

CTA:  
What should I do next?

---

# Development Philosophy

Build reusable components.

Prefer refinement over redesign.

Keep the codebase modular, maintainable, and scalable.

Avoid unnecessary complexity.

Every new component should be reusable elsewhere in the application.

---

# Cursor Workflow

ChatGPT responsibilities:

- Product strategy
- UX
- Information architecture
- Brand positioning
- Design reviews
- Prompt engineering

Cursor responsibilities:

- Component implementation
- Refactoring
- Styling
- Code generation
- File editing

Guidelines:

- One prompt = one responsibility.
- Review after every major implementation.
- Prefer multiple focused prompts over one large prompt.

# Repository Structure

Kamilyan/

- website/
- docs/
- notes/
- assets/

The website directory contains the application.

The docs directory contains the long-term project knowledge.

The notes directory contains temporary ideas and brainstorming.

The assets directory contains design resources, imagery, and branding materials.

---

# Definition of Done

A feature is complete when:

- It supports the Kamilyan brand.
- It communicates clearly.
- It is visually polished.
- It is accessible.
- It is responsive.
- It uses clean, maintainable code.
- It is reusable where appropriate.
